# MySQL Queries Guide - DineMaster Auth Service

## How to View SQL Queries

The auth-service is configured to show SQL queries in the console:
- `spring.jpa.show-sql=true` in `application.properties`

To view the queries:
1. Start the auth-service backend: `cd backend/auth-service && mvn spring-boot:run`
2. Look at the console output - all SQL queries will be displayed

## Database Schema

### Users Table
```sql
CREATE TABLE users (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    name VARCHAR(255) NOT NULL,
    phone VARCHAR(255),
    birthday VARCHAR(255),
    role VARCHAR(255) NOT NULL,
    loyalty_points INT,
    birthday_offer_years VARCHAR(255),
    is_verified BOOLEAN NOT NULL DEFAULT FALSE,
    created_at DATETIME
);
```

### Verification Tokens Table
```sql
CREATE TABLE verification_tokens (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    token VARCHAR(255) NOT NULL,
    expiry_date DATETIME NOT NULL,
    user_id BIGINT NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## Common SQL Queries Generated by Hibernate

### 1. Find User by Email
**Repository Method:** `userRepository.findByEmail(email)`
**Generated SQL:**
```sql
SELECT u.id, u.email, u.password, u.name, u.phone, u.birthday, 
       u.role, u.loyalty_points, u.birthday_offer_years, u.is_verified, u.created_at
FROM users u
WHERE u.email = ?;
```

### 2. Check if Email Exists
**Repository Method:** `userRepository.existsByEmail(email)`
**Generated SQL:**
```sql
SELECT CASE WHEN COUNT(u.id) > 0 THEN true ELSE false END
FROM users u
WHERE u.email = ?;
```

### 3. Check if Phone Exists
**Repository Method:** `userRepository.existsByPhone(phone)`
**Generated SQL:**
```sql
SELECT CASE WHEN COUNT(u.id) > 0 THEN true ELSE false END
FROM users u
WHERE u.phone = ?;
```

### 4. Create New User (Registration)
**Repository Method:** `userRepository.save(user)`
**Generated SQL:**
```sql
INSERT INTO users (email, password, name, phone, birthday, role, 
                  loyalty_points, birthday_offer_years, is_verified, created_at)
VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?);
```

### 5. Update User
**Repository Method:** `userRepository.save(user)`
**Generated SQL:**
```sql
UPDATE users
SET email = ?, password = ?, name = ?, phone = ?, birthday = ?,
    role = ?, loyalty_points = ?, birthday_offer_years = ?,
    is_verified = ?, created_at = ?
WHERE id = ?;
```

### 6. Find User by ID
**Repository Method:** `userRepository.findById(id)`
**Generated SQL:**
```sql
SELECT u.id, u.email, u.password, u.name, u.phone, u.birthday, 
       u.role, u.loyalty_points, u.birthday_offer_years, u.is_verified, u.created_at
FROM users u
WHERE u.id = ?;
```

### 7. Find All Users
**Repository Method:** `userRepository.findAll()`
**Generated SQL:**
```sql
SELECT u.id, u.email, u.password, u.name, u.phone, u.birthday, 
       u.role, u.loyalty_points, u.birthday_offer_years, u.is_verified, u.created_at
FROM users u;
```

### 8. Delete User
**Repository Method:** `userRepository.deleteById(id)`
**Generated SQL:**
```sql
DELETE FROM users
WHERE id = ?;
```

## Verification Token Queries

### 1. Create Verification Token
**Repository Method:** `verificationTokenRepository.save(token)`
**Generated SQL:**
```sql
INSERT INTO verification_tokens (token, expiry_date, user_id)
VALUES (?, ?, ?);
```

### 2. Find Token by Token Value
**Repository Method:** `verificationTokenRepository.findByToken(token)`
**Generated SQL:**
```sql
SELECT vt.id, vt.token, vt.expiry_date, vt.user_id
FROM verification_tokens vt
WHERE vt.token = ?;
```

### 3. Find Token by User
**Repository Method:** `verificationTokenRepository.findByUserId(userId)`
**Generated SQL:**
```sql
SELECT vt.id, vt.token, vt.expiry_date, vt.user_id
FROM verification_tokens vt
WHERE vt.user_id = ?;
```

## How to Run Queries Directly in MySQL

### Connect to MySQL
```bash
mysql -u root -p
```

### Select Database
```sql
USE dinemaster_auth;
```

### View All Users
```sql
SELECT * FROM users;
```

### View Specific User
```sql
SELECT * FROM users WHERE email = 'user@example.com';
```

### View Verification Tokens
```sql
SELECT * FROM verification_tokens;
```

### Count Users by Role
```sql
SELECT role, COUNT(*) as count
FROM users
GROUP BY role;
```

### View Unverified Users
```sql
SELECT * FROM users WHERE is_verified = false;
```

### View Users with Loyalty Points
```sql
SELECT email, name, loyalty_points
FROM users
WHERE loyalty_points > 0
ORDER BY loyalty_points DESC;
```

## Hibernate DDL Auto Modes

The application uses `spring.jpa.hibernate.ddl-auto=update` which:
- Automatically creates tables if they don't exist
- Updates table structure if the entity model changes
- Does NOT drop existing data

Other modes:
- `none` - No automatic schema generation
- `validate` - Validate schema, make no changes
- `create` - Create schema, destroys previous data
- `create-drop` - Create schema on startup, drop on shutdown
- `update` - Update schema without destroying data (current setting)

## Performance Tips

1. **Add Indexes** for frequently queried columns:
```sql
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_phone ON users(phone);
CREATE INDEX idx_users_role ON users(role);
```

2. **Use Native Queries** for complex operations:
```java
@Query(value = "SELECT * FROM users WHERE created_at > :date", nativeQuery = true)
List<User> findUsersCreatedAfter(@Param("date") LocalDateTime date);
```

3. **Enable Query Logging** (already enabled):
```properties
spring.jpa.show-sql=true
logging.level.org.hibernate.SQL=DEBUG
logging.level.org.hibernate.type.descriptor.sql.BasicBinder=TRACE
```
